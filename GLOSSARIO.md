# 📖 Glossário de QA     

Termos e conceitos estudados durante minha jornada em QA.

## B

**Bug** — Defeito ou comportamento inesperado em um sistema.

**Bug Report** — Documento que descreve um bug encontrado,
com passos para reproduzir, resultado esperado e obtido.

## C

**Caso de Teste (Test Case)** — Roteiro criado antes de testar,
descrevendo o que testar e qual resultado esperar.

**Critério de Aceite** — Define o que é necessário para
considerar uma funcionalidade aprovada.

## E

**Environment (Ambiente)** — Local onde os testes são executados,
como desenvolvimento, homologação ou produção.

## P

**Pré-condição** — O que precisa estar pronto antes
de executar um teste.

## R

**Regressão** — Teste que verifica se algo que funcionava
continua funcionando após mudanças no sistema.

## S

**Severidade** — O quão grave é um bug tecnicamente.

**Smoke Test** — Teste rápido para verificar se o
básico do sistema está funcionando.

## T

**Test Plan** — Documento que organiza toda a
estratégia de testes de um projeto.

**Git** — Sistema de controle de versão que rastreia 
alterações nos arquivos do projeto.

## A

**API** (Application Programming Interface) — Interface de 
comunicação entre sistemas. Permite que aplicações se 
comuniquem entre si.

## C

**CRUD** — Conjunto das 4 operações básicas de uma API:
Create (POST), Read (GET), Update (PUT), Delete (DELETE).

## J

**JSON** (JavaScript Object Notation) — Formato padrão 
de resposta das APIs. Organiza dados em chave e valor.

## M

**Método HTTP** — Tipo de operação enviada para uma API.
Principais: GET, POST, PUT, DELETE.

## S

**Status Code** — Código numérico que indica o resultado 
de uma requisição. Ex: 200 OK, 201 Created, 404 Not Found.
